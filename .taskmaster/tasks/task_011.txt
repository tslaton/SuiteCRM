# Task ID: 11
# Title: Implement Upcoming Showings/Tasks Widget
# Status: pending
# Dependencies: 1, 8
# Priority: medium
# Description: Create a widget for the Real Estate Hub that displays upcoming showings and important tasks.
# Details:
Create a custom dashlet class (custom/modules/Home/Dashlets/UpcomingDashlet/UpcomingDashlet.php) that extends DashletGeneric. Implement a query to fetch upcoming meetings (showings) and tasks due in the next 7 days. Create a Smarty template that displays these items in a timeline or list format. Group items by day (Today, Tomorrow, This Week). Include key details for each item (time, property/client, type). Add quick-action buttons to complete tasks or reschedule meetings. Implement color coding based on priority or proximity to due date. Add a 'View Calendar' link to the full calendar view. Include a count of overdue items with a warning indicator. Filter to focus on property showings and transaction-related tasks by default. Use this query pattern for meetings: SELECT m.id, m.name, m.date_start, m.status, p.name as property_name FROM meetings m LEFT JOIN meetings_properties_c mp ON m.id = mp.meeting_id LEFT JOIN properties p ON p.id = mp.property_id WHERE m.assigned_user_id = '{$current_user->id}' AND m.date_start > NOW() AND m.date_start < DATE_ADD(NOW(), INTERVAL 7 DAY) ORDER BY m.date_start ASC

# Test Strategy:
Test the widget with various combinations of upcoming meetings and tasks. Verify items are grouped correctly by day. Test quick-action buttons functionality. Verify overdue items are highlighted appropriately. Test with items that have and don't have related properties. Verify the 'View Calendar' link works correctly.
