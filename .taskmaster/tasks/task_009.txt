# Task ID: 9
# Title: Implement Active Listings Widget
# Status: pending
# Dependencies: 1, 8
# Priority: medium
# Description: Create a widget for the Real Estate Hub that displays the agent's active property listings.
# Details:
Create a custom dashlet class (custom/modules/Home/Dashlets/ActiveListingsDashlet/ActiveListingsDashlet.php) that extends DashletGeneric. Implement a query to fetch properties with status='Active' related to the current user. Create a Smarty template for the dashlet that displays properties in a card format with key details (address, price, days on market, thumbnail image). Add sorting options (newest, price high/low). Include quick-action buttons for each listing (edit, view, schedule showing). Implement pagination if there are many listings. Add a 'View All' link to the full Properties list view with an active filter applied. Use this query pattern: SELECT p.id, p.name, p.price_c, p.address_c, p.bedrooms_c, p.bathrooms_c, p.square_footage_c, p.days_on_market_c FROM properties p JOIN properties_cstm pc ON p.id = pc.id_c WHERE p.assigned_user_id = '{$current_user->id}' AND pc.status_c = 'Active' ORDER BY p.date_entered DESC LIMIT 5

# Test Strategy:
Test the widget with various numbers of active listings. Verify all property information displays correctly. Test sorting functionality. Verify quick-action buttons work as expected. Test pagination if implemented. Verify the 'View All' link applies the correct filter to the list view.
